# AI Storyboard Director v5.2

> 导演级 AIGC 分镜系统：把剧本拆成有视觉概念、有剪辑动机、可执行、可审查的 AG-CLIP 镜头链。

![Version](https://img.shields.io/badge/version-v5.2-blue.svg)
![AIGC](https://img.shields.io/badge/AIGC-video%20storyboard-orange.svg)
![License](https://img.shields.io/badge/license-MIT-green.svg)

## 这个技能适合谁

- AIGC 视频创作者：把剧本变成可投喂视频生成工具的镜头方案。
- 短剧/短片导演：先快速拿到设计稿，再展开可执行生产稿。
- 提示词工程师：把分镜结果继续转成即梦、可灵、Runway、Pika、Seedance 等工具的提示词。
- Agent 工作流开发者：把它作为“剧本理解 → 分镜设计 → 提示词生成”的导演模块。

## 三个文件怎么选

| 文件 | 用途 | 适合场景 |
|---|---|---|
| `ai-storyboard-director5.2.skill` | 可安装技能包，内含主技能、知识库和示范资产 | Codex/Agent 技能环境 |
| `ai-storyboard-director5.2-全量单文件.md` | 完整自包含版，内联主指令、知识库和示范集 | 长上下文模型、正式分镜、需要完整知识深度 |
| `ai-storyboard-director5.2-精编单文件.md` | 小上下文精编版，保留输出契约、铁律、受控词表和核心决策程序 | 快速草案、小模型、轻量工作流 |

## 快速上手

### 方案 A：使用技能包

1. 导入 `ai-storyboard-director5.2.skill`
2. 发送剧本
3. 输入：

```text
请按 AI Storyboard Director v5.2 默认流程拆分镜。先给设计稿，我确认后再展开生产稿。
```

### 方案 B：使用单文件版

1. 打开全量版或精编版 Markdown
2. 复制全文作为系统提示词或对话首条消息
3. 发送剧本并输入：

```text
拆分镜。默认先出设计稿，设计稿通过后再出生产稿。
```

### 方案 C：审查已有分镜

```text
请进入审查模式，检查这份分镜的连续性、导播病、不可生成描述、剪辑动机和输出契约问题，并给出修正版。
```

## v5.2 核心更新

- **立概念**：切节拍前先锁定视觉概念、母题、空间权力几何、反常规决定和风格坐标。
- **双速交付**：设计稿给人读，生产稿给机器执行。
- **导播病检查**：识别通篇默认拍法、缺少视觉主意的分镜。
- **创造性四查**：导播测试、概念三镜引证、反常规执行对账、母题链完整性。
- **兼容旧契约**：生产稿字段与 v4.5/v5.0/v5.1 保持兼容。

## 输出会长什么样

### 设计稿

设计稿用于快速判断方向是否对，通常包含：

- 视觉概念
- 母题建立、变奏、打破
- 空间权力几何
- 反常规决定
- 风格坐标
- 薄镜头表
- 设计稿速检

### 生产稿

生产稿用于下游执行，重点是：

- 每镜一个连续 AG-CLIP
- 入口引用和出口状态
- 景别、机位、运镜、光色、声音
- 内部时间线
- 剪辑动机
- 翻车点和规避
- 全项自检

## 推荐提示

```text
下面是剧本。请先读懂情节、人物和情绪终点，再立概念。
默认先输出设计稿，不要直接展开生产稿。

<剧本>
```

```text
我确认设计稿。请按 v5.2 生产稿契约展开，全字段输出，注意只写物理事实，不写导演散文。
```
